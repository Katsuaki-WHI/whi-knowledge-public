# 読み物・ドキュメントのバージョン管理ルール

レポート・ガイド・仕様書など、**読み物・ドキュメント類は必ずバージョン管理する**。
口頭の決定や日付だけの管理では、同日改訂・複数系統を区別できず、どの版が最新・正かを見失う。

## 原則

1. **改訂時は必ず版番号を付与する**
   - 「最新版」「修正版」のような曖昧な呼称で運用しない。明示的な版番号（v2.1 / v2.3 / Public v1.0 等）を付ける。

2. **版番号はファイル名にも入れる**
   - 日付だけだと**同日改訂を区別できない**。ファイル名に版番号を入れる。
   - md・docx・図など派生物すべてに同じ版番号を付ける。

3. **系統が違うものは別番号体系で管理し、ファイル名で区別できるようにする**
   - 例：内部版＝v系（v2.1 → v2.3）／外部公開版＝P系（Public v1.0）。
   - ファイル名にも系統と版を入れる。同名で中身違いを作らない。

4. **複数系統がある場合は対応関係を記録に残す**
   - どの内部版から、どの公開版を作ったかを記録に明記する（例：「Public v1.0 は内部版 v2.3 を基に作成」）。
   - 系統間で共有する数値・主張・出典は、片方を直したらもう片方も揃える（整合の責任を残す）。

5. **版を上げる節目に何が変わったかを記録する**
   - 「この版で何を変えたか・数値や主張が変わったか不変か」を残す。
   - 数値・出典・主張が変わる改訂は特に明記する（後から「いつ・なぜ変わったか」を追えるように）。

## 背景

内部版と外部提出用の公開版を並行管理する際に確立。当初、公開版のファイル名に版番号が無く内部版と一目で区別できなかったため、ファイル名に版番号を付与し（履歴を保持してリネーム）、2系統管理を明確化した。
