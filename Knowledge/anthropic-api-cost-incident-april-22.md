# Anthropic APIコスト$27.50事故（2026/04/22）

## 事象

2026/04/22のテスト中、バグによる連打で1日に**$27.50**を消費。
通常運用の想定を大幅に超える異常なコスト発生。

## 原因

1. **エラー時の連打**：API呼び出し失敗時に再実行を繰り返した
2. **Spend Limit未設定**：上限なしの状態だった
3. **レート制限未実装**：ユーザー単位の上限なし
4. **連打防止未実装**：クールダウン期間なし

## 再発防止策

### 1. Anthropic Console設定
- Spend Limit：**$200-300/月**
- アラート閾値：**50% / 80% / 95%**
- 日次でUsage画面を確認する習慣

### 2. アプリケーション側のレート制限
- **1ユーザー1日$3上限**
- **ラリー上限44/セッション**（[[2026-04-22-rally-cap-44-per-session]]）
- **2秒クールダウン**（連続呼び出し防止）

### 3. ハイブリッドモデル戦略
- Week0配信・リマインド：Haiku（軽量・低コスト）
- 振り返り生成：Sonnet（バランス型）
- 物語の書（本編）：Opus（高品質・高コスト）

### 4. 強制着地での完結
- 上限到達時はブロックではなく強制着地（[[2026-04-22-rate-limit-forced-landing]]）
- $3.04に収まる設計

## 教訓

- **「テスト中だから」と油断しない**：本番と同じコストが発生する
- **エラー時こそ立ち止まる**：原因究明前に再実行しない
- **上限設定は必須**：技術的・経済的セーフティネット
