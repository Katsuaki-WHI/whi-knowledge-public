# API費用管理

## 教訓

過去にAPIバグの再現確認のため連打した結果、予期せぬコスト消費が発生した。
従量課金APIは、デバッグ・連打・無限ループで急激にコストが跳ねる。

## 必須設定

### Anthropic API
- Spend Limit を月$200-300 で設定
- アラート閾値を50%・80%・100% で設定
- 異常検知時に即座にメール通知

### ハイブリッドモデル戦略（コスト最適化）

なかまクエストでのモデル使い分け方針：
- **Week0配信・リマインド**：Haiku（軽量・低コスト）
- **振り返り生成**：Sonnet（バランス型）
- **物語の書（本編）**：Opus（高品質・高コスト）

用途に応じてモデルを切り替え、Opusの使用を必要最小限に絞る。

## 連打防止

- APIエラー発生時、原因究明前に再実行を繰り返さない
- 同じリクエストを5回以上失敗したら手動介入
- 自動リトライは最大3回・指数バックオフを必ず実装
