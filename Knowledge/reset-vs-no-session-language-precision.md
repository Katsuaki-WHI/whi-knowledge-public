# 「リセット済み」vs「最初から無い」の言葉の精度

## 問題

テスター連絡で「リセット済み」と書いたが、実際は「最初からセッションが存在していなかった」。
言葉の精度が状況を誤認させた事例。

## 対策

### 1. DBの実状態を確認してから連絡を書く
- Claude CodeでDBクエリ実行
- 「実際の状態」を確認してから言語化

### 2. 3つの状態を区別する

| 表現 | 意味 |
|---|---|
| リセットした | セッションが存在していたが、状態を初期化した |
| 最初から無かった | セッション自体が存在していない |
| 再発行した | 既存のものを破棄して新規発行した |

### 3. バイブル本体とコーチングセッションは別物
- バイブル＝レポート本体
- コーチングセッション＝対話履歴
- 「バイブルはあるがコーチングはまだ」のようなケースを明確に区別
