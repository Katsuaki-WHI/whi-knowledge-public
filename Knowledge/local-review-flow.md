# ローカル確認の標準フロー（非エンジニアのユーザーによる目視レビュー）

push 前に、ユーザーがブラウザで実物を確認するための標準フロー。
「push禁止・ローカルまで」の指示や、見た目・UXの変更を含むバッチで使う。

## 標準フロー

1. **Claude Code が dev サーバーを起動し、起動を維持する**
   - 例：`npm run dev`（バックグラウンド）。確認が終わるまで止めない。
   - build と dev で出力ディレクトリ（Next.js の `.next` 等）が競合し得る。build 後に dev を使うときは
     キャッシュを消してから起動するとキャッシュ破損による 500 を避けられる。

2. **確認用URLを1行プレーンテキストで提示する（表に入れない）**
   - 表組みに入れるとトークン付きURLが折り返してコピー不可になる（[[mistakes]] 参照）。
   - ラベルは別行、URLは単独の1行で。

3. **ユーザーがブラウザで確認する**（日英・スマホ幅/PC幅・エラーの有無など）。

4. **判定 → 次のアクション**
   - OK → ユーザーが「止めて」→ push 指示（dev 停止 → commit → push）。
   - NG → 修正して 1〜3 を繰り返す。

## 注意

- **ポート番号は提示された実ポートを使う**。dev サーバーが既定ポートを取れないと別ポートに
  フォールバックする。古いポート番号と混同しない。
- **実ブラウザ検証はヘッドレスブラウザ（playwright-core＋システムChrome）で代替可能**。
  `--no-save` で入れれば依存定義を汚さず L4（実HTTP・実描画）検証ができる。
  dev は HMR の常時接続で `networkidle` に到達しないため、`domcontentloaded`＋明示的な
  要素待ち（waitForSelector）を使う。本番URLにも同じ方法でスモークできる。
- 本番デプロイ確認はデプロイ状態が「Ready」になるのを待ってからスモークする。
