# mistakes.md - 再発防止の核

かつさんから明示的に訂正された、再発しうるパターンを記録。
新規追加は AI-RULES.md の「3. mistakes.md への追記ルール」3条件を満たすときのみ。

---

## 2026-05-19：Build PASS は本番OKを意味しない

**NG**：`npm run build` が成功したら「実装完了」と報告する
**Correct**：ビルド成功後、必ずブラウザベースの実HTTPリクエストフローで動作確認してから「実装完了」と報告する
**Trigger**：Next.js/RSC（React Server Components）を使うプロジェクトでの実装完了報告時。
RSCマーシャリング違反はビルド成功でも本番でエラーになるため、ビルド成功だけでは品質を担保できない。

## 2026-05-19：「調査済み・正常」報告を鵜呑みにしない

**NG**：Claude Codeが「動作確認しました、正常です」と報告したら、それで完了扱いにする
**Correct**：ブラウザベースの実HTTPリクエストフローで、かつさんもしくはClaude Code自身が実際に画面・APIを叩いて確認する
**Trigger**：実装完了報告・バグ修正完了報告のとき。
過去に「正常」報告のあと本番で再現した事例あり。

## 2026-05-19：APIエラー時の連打でコストが跳ねる

**NG**：APIエラー発生時、原因究明前に再実行を繰り返す
**Correct**：エラー発生時はまず原因究明。Spend Limit設定とアラート閾値を必ず設定。連打しない。
**Trigger**：Anthropic API・OpenAI APIなど従量課金APIのデバッグ時。
過去に予期せぬコスト消費が発生した。詳細は [[api-cost-management]]。

**NG**：「削除して」「本番デプロイして」を即実行
**Correct**：実行前に必ずユーザーに確認。「本当に削除して良いですか？」「本番に反映しますか？」と一拍置く。
**Trigger**：削除コマンド・本番デプロイ・DBスキーマ変更・データ移行など、戻せない操作。

## 2026-05-20：マスター移行時は旧参照を全使用箇所grepで総点検する

**NG**：マスターデータを新旧で移行するとき、一部のセクション・コンポーネントだけ
旧マスター参照を残す
**Correct**：移行時は旧参照（旧関数名・旧配列名・.find()等）を src/ 全体で grep し、
全使用箇所を一覧化してから移行する。移行漏れがあると、旧マスターに無い値を引いた
特定ユーザーだけ表示が欠ける（他ユーザーは正常なので気づきにくい）。
**Trigger**：マスターデータの新旧移行時・getXxx系の参照関数を差し替えるとき。
**実例**：なかまクエスト「お主のキャラクター」セクションが getCharacter()（旧21件マスター）
を参照したままで、C_貢献（新126件マスターのみの値）を持つメンバーだけカードが空白になった。
原因はデータ欠損ではなくコードの移行漏れ。getCharacterById()（126件対応）へ切替で解決。

## 2026-05-27：マスターのキー命名規則は1つに統一する（4分裂が同種バグを量産）

**NG**：マスターごとにキーの大文字小文字・balance表記（balance/DISC）がバラバラだと、
判定の出力キーとマスターのキーが一致せず、特定タイプだけ表示が欠ける。
**Correct**：判定の出力キーとマスターのキーの命名規則を1つに統一する。
consumer側で都度正規化（.toUpperCase()等）を覚える設計はバグの温床。
**Trigger**：タイプキーでマスターを引くすべての箇所。新マスター追加時。
**実例**：なかまサーベイで computeIndividualTypeId はMIXED case（iCS等）を出すのに
GAP_COMMENTSはUPPERCASE（ICS等）で、15/21タイプのギャップ予告が非表示になった。
これは「お主のキャラクター」非表示バグ（2026-05-20記録）と同じクラスの再発。
根本原因はマスターキー命名規則の4分裂。

## 2026-05-19：knowledge baseのバージョン更新時、Project手順の同期を忘れない

**NG**：whi-knowledge/Preferences/conversation-rules.md のバージョンを v2.1 → v2.2 に更新したのに、claude.ai Project「手順」内のバージョン表記（「会話ルール v2.1」「20260519_WHI_conversation_rules_v2.1.md」等）を更新し忘れる

**Trigger**：会話ルールのバージョンを更新したとき・Project手順内に明示的なバージョン表記があるとき
**出典**：whi-knowledge構築（2026/05/19）
