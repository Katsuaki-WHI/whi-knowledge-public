# 多言語対応・i18n戦略

## 本質

**Webアプリは常に多言語対応への発展可能性を担保する。**

WHIはグローバル展開する戦略のため、最初から「いつでも何言語にでも増やせる」状態にする。

## 日英両対応マスト

- 全画面・全コンテンツ日英両対応必須（片方だけの更新禁止）
- **日英同時開発は初期段階のマスト**
- 後付けでi18n化するのは技術負債を生む
- 英語版が主・日本語版が従（テストマーケット）の位置づけ
- 英語版は現代的で読みやすい1バージョンのみ（王様口調・古英語等なし）

## 実装ルール（Claude Code向け絶対遵守）

### 1. 文言のハードコーディング禁止
コード中に「送信する」など表示文字を直接書かない。

### 2. 辞書ファイル分離
日本語は`ja.json`、英語は`en.json`などに分離。
コードは辞書のキー（例：`submit_button`）を参照する形にする。

```typescript
// NG
<button>送信する</button>

// OK
<button>{t('submit_button')}</button>
```

### 3. locale切替可能な構造
ユーザーの言語設定（ブラウザ言語・ユーザー選択）に応じて、自動で辞書を切り替えられる仕組みで実装する。

### 4. 3言語以上への拡張に耐える設計
将来（中国語・韓国語・スペイン語・ベトナム語など）の追加が「辞書ファイル1個追加」で済む構造にする。

## かつさんの確認ポイント

新機能の実装報告を受けたとき、以下を Claude Code に確認：
- ハードコーディングしていないか
- 辞書ファイル経由で文字を扱っているか
- locale切替が機能するか
