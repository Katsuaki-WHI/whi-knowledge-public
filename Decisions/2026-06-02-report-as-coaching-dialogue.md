# レポートは「診断結果」でなく「対話の出発点」（公開可能な抽象原則）

サーベイのレポート設計についての一般原則。具体的な分析数値・指標の構成・
事業モデル・価格は含まない（それらは非公開）。ここに残すのは、他プロダクトにも
転用できる設計思想のレベルのみ。

## 原則

1. **レポートは診断でなく対話の出発点**
   結果を「正解」として突きつけるのではなく、「ここから何を話す?」という
   対話のキャンバスとして提示する。自分で気づくから人は動く。

2. **メタファー（絵）で解釈を本人に返す**
   数字だけで出すと「高い/低い」の相対論で思考が完結・平準化しやすい。
   絵やメタファーに変換すると、本人が自分の文脈で意味づけでき、オーナーシップが生まれる。
   「分かりにくくする」のではなく「解釈の余地を残し、思考を本人に返す」設計。

3. **レポートの人格は「応援」だけでなく「問いかけ」**
   「あなたはできる」と主導するだけでなく、「なぜだと思う?」と思考を本人に渡す。
   問いを立てられた人は、自分で答えを探したくなる。

4. **事実と言葉で役割を分ける**
   ビジュアル（色・数値・図）は事実を正直に見せる（ごまかさない）。
   言葉（コメント・問い）で勇気づけ、現実を直視する勇気を支援する。
   嘘やおべっかで気持ちよくさせない。自分で良くしようとする動機づけが本質。

5. **個人向けと集団向けで語り口を変える**
   個人＝一人の見え方を肯定し勇気づける（承認・スポンサーシップ）。
   集団＝答えは対話の中にあるとして問いかける。
   見かけが同じ図でも、記す物語は別物にする。

6. **補助指標は主役でなく「全体像の一部の拡大鏡」**
   ダッシュボードや個別スコアは、主役（全体像）の一部を拡大して見せるもの。
   目線を全体像へ収束させる補助に徹し、主役を脇に追いやらない。

## 最上位の判断基準
迷ったら「関わる全員のパフォーマンスとやりがいが、最も高まる方を取る」。
犯人探しの道具にしない。成功循環モデル（関係性→思考→行動→結果）を正回転させる。
